CRF

순서가 있는 데이터에서 각 위치의 라벨을 예측할 때 사용하는 모델

토큰 하나만 따로 보는 게 아니라, 앞뒤 라벨 관계까지 같이 고려해서 가장 자연스러운 라벨 시퀀스를 찾는 모델

CRF는 Emission score, Transition score 두가지 점수를 봄

CRF의 핵심은 문장 전체를 보고 라벨을 정함

CRF장점

라벨 간 관계를 반영할 수 있음, 라벨 간 관계를 반영할 수 있음, 해석이 비교적 쉬움, 전통적인 NER baseline으로 강력함

CRF단점

span 자체를 직접 예측하지는 않음, 중첩 엔티티 처리 어려움, 최신 span 기반 모델보다 유연성이 떨어질 수 있음

CRF를 단독으로 쓰기보다 BERT 같은 인코더를 같이 씀

CRF는 보통 BIO/BIOES 태깅과 같이 사용

CRF를 해석이 쉬운 이유

토큰 점수와 라벨 연결 점수를 따로 볼 수 있음, 전이 행렬을 직접 볼 수 있음, 잘못된 예측도 원인 추적이 쉬움
